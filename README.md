# AES-128 Hardware Accelerator with APB Interface

## 🏗️ Top-Level Architecture

The following diagram illustrates the overall architecture of the AES-128 hardware accelerator integrated with an AMBA APB interface. The processor communicates with the AES core through APB transactions, enabling secure key and data transfer.

<img width="723" height="439" alt="aes_apb" src="https://github.com/user-attachments/assets/dfb4924e-0f8f-4ab0-aaed-d0bdb2c2765c" />
### 🔍 Description

The system consists of the following major components:

- **Processor (APB Master)**  
  Initiates read/write transactions to configure and control the AES core.

- **APB Interface**  
  Handles communication using APB signals such as PADDR, PWDATA, PRDATA, PSEL, PENABLE, and PWRITE.

- **AES Core**  
  Performs AES-128 encryption and decryption operations.

- **Control Unit (FSM)**  
  Generates control signals for AES rounds and manages operation flow.

- **Datapath**  
  Implements AES transformations including:
  - SubBytes
  - ShiftRows
  - MixColumns
  - AddRoundKey

- **Key Expansion Unit**  
  Generates round keys required for encryption and decryption.

---

### 🔁 Data Flow

1. Processor writes key and plaintext via APB.
2. AES core processes data through multiple rounds.
3. Ciphertext is generated and stored.
4. Processor reads output via APB interface.


## 🔐 AES-128 Encryption Flow

The following diagram illustrates the internal transformation steps of the AES-128 encryption process. The algorithm operates on 128-bit data blocks and performs a sequence of round-based transformations.
<img width="753" height="1065" alt="Block-Diagram" src="https://github.com/user-attachments/assets/6bcf02ca-ea58-4345-a3b1-be6082a298e6" />
### 🔍 Description

AES-128 operates on a 128-bit plaintext organized as a **state matrix** and applies multiple transformation rounds using a secret key.

---

### 🔄 Encryption Steps

1. **Initial Round**
   - AddRoundKey is applied using the original key.

2. **Main Rounds (9 Rounds)**
   Each round consists of:
   - **SubBytes** → Non-linear substitution using S-box
   - **ShiftRows** → Row-wise permutation
   - **MixColumns** → Column-wise mixing (diffusion)
   - **AddRoundKey** → XOR with round key

3. **Final Round**
   - SubBytes
   - ShiftRows
   - AddRoundKey  
   *(Note: MixColumns is not used in the final round)*

---

### 🔑 Key Expansion

- The initial 128-bit key is expanded into **round keys**
- Each round uses a unique key generated by the key expansion unit

---

### 📌 Output

- Final output is the **128-bit ciphertext**
